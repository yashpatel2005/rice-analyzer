# Architecture and Concepts Report: Rice Analyzer

This document outlines the architectural design and the scientific concepts driving the **Rice Analyzer**, an advanced computer vision system for morphometric analysis and quality grading of rice grains.

---

## 1. System Architecture Overview

The system is built as a **Hybrid Web Application** utilizing a decoupled frontend-backend architecture:

- **Frontend (Vercel)**: A static Progressive Web App (PWA) built with Vanilla JavaScript, HTML, and Bootstrap. It handles user interaction, local WebRTC device camera capture, and responsive data visualization using Chart.js.
- **Backend (Python Flask via SSH/Cloudflare)**: A stateless RESTful API hosted on a dedicated Linux server. It processes high-resolution images via OpenCV and scikit-image, executes the computer vision pipelines, and returns structured JSON telemetry.

### Core Modules
The backend is modularized into discrete responsibilities located in the `core/` directory:
1. `preprocessing.py`: Image normalization, noise reduction, and polarity detection.
2. `segmentation.py`: Classical Watershed-based isolation of individual grains.
3. `clustering_segmentation.py`: Advanced unsupervised ML isolation using K-Means and GrabCut.
4. `measurement.py`: Morphometric feature extraction (length, width, area, aspect ratio).
5. `statistics.py`: Aggregation and variance calculations (KDE distributions).
6. `classification.py`: Geometric categorization (Long, Medium, Short grain).
7. `quality.py`: Defect detection (broken, chalky) based on area and intensity thresholds.
8. `grading.py`: Market-grade calculation based on defect percentages.

## 2. Core Concepts & Algorithms Used

The analyzer relies heavily on mathematical morphology and unsupervised learning rather than pre-trained deep learning networks. This ensures the system is highly deterministic, fast, and requires zero external training data.

### A. Dynamic Polarity & Illumination Equalization
- **CLAHE (Contrast Limited Adaptive Histogram Equalization)**: Before any thresholding occurs, the image undergoes CLAHE. Unlike global histogram equalization which can wash out details, CLAHE operates on small local tiles (e.g., 8x8), enhancing the contrast of translucent grains against the background while preventing noise amplification in uniform areas.
- **Polarity Detection**: The system calculates the median intensity of the image borders versus the center to mathematically deduce if the image is "bright grains on a dark background" or vice versa, dynamically inverting the binary mask if necessary.

### B. Classical Segmentation (Watershed)
- **Distance Transform**: The binary image is converted into a topographical map where the brightness of each pixel corresponds to its distance from the nearest background pixel. The centers of the grains become "peaks."
- **Canny Edge Markers**: To prevent two touching grains from being merged into one, the system uses the Canny Edge Detection algorithm to find high-gradient boundaries. These edges guide the peak-finding logic.
- **Watershed Algorithm**: The system simulates "flooding" the topographical map from the peaks. Where the "water" from two different peaks meets, a boundary line is drawn, perfectly separating touching grains.

### C. Advanced Unsupervised Segmentation (K-Means + GrabCut)
For complex environments (e.g., heavily shadowed or highly clustered grains), the system utilizes a two-step unsupervised machine learning approach:
1. **K-Means Color Clustering**: The image is transformed from the RGB color space to the **HSV (Hue, Saturation, Value)** space. K-Means (with K=2) mathematically clusters all pixels into two groups based on their 3D color coordinates. This groups all "rice-colored" pixels together, entirely ignoring shadows.
2. **GrabCut Refinement**: The rough mask generated by K-Means is fed into GrabCut as a "Probable Foreground." GrabCut builds a Gaussian Mixture Model (GMM) of the color distributions and uses a Min-Cut/Max-Flow graph algorithm to perfectly refine the sub-pixel edges of the grains.

### D. Morphometric Feature Extraction
Once a grain is isolated as a binary contour, mathematical moments are calculated:
- **Spatial Moments ($M_{ij}$)**: Used to calculate the centroid (center of mass) of the grain.
- **Image Orientation**: By calculating the central moments ($\mu_{20}, \mu_{02}, \mu_{11}$), the system determines the angle of the grain's major axis.
- **Minimum Enclosing Rectangle**: A rotated bounding box is fitted around the contour using the Rotating Calipers algorithm. The dimensions of this box dictate the true **Length** and **Width** of the grain, regardless of its orientation in the image.

### E. Calibration (Pixels to Millimeters)
The system relies on a physical calibration constant (`pixels_per_mm`). 
- When an image of a known object (e.g., a coin or a checkerboard) is uploaded, the system detects the object's pixel width and divides it by the real-world physical width to generate the conversion ratio. 
- Alternatively, the system attempts a fallback heuristic by estimating the camera's Field of View (FOV) using EXIF Focal Length data.

### F. Quality Grading Engine
The system evaluates the extracted features against strict threshold logic:
- **Broken Grains**: Any grain whose area is significantly lower than the statistical median area (e.g., $< 60\%$) or whose aspect ratio falls below the threshold for its classified type.
- **Market Grading**: Based on the percentage of broken grains (e.g., Grade A requires $< 5\%$ broken, Grade B $< 15\%$, etc.), dynamically categorizing the batch's commercial value.
